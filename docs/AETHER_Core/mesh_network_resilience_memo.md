# AETHER Web-Lite: メッシュ・ネットワーク耐性調査メモ

**作成日**: 2026-03-20
**対象**: 10,000ノード規模の疎結合メッシュ（Degree=6）

## 1. シミュレーション結果のサマリー

### 10,000ノード規模での段階的離脱テスト
生存者の再構築（ヒーリング）ロジックが常に稼働している状態でのシミュレーション結果。

| 離脱率 | 残存ノード数 | ネットワークの健全性 |
| :--- | :--- | :--- |
| **初期** | 10,000人 | **100% 健全** (単一メッシュ) |
| **50% 減少** | 4,736人 | **99.5% 健全** (数名の孤立のみ) |
| **同時30%死** | (激変時) | **99.9% 健全** (自己修復成功) |
| **同時50%死** | (パニック時) | **96.3% 健全** (一部の孤立が発生) |

### 結論
- **自己修復の有効性**: ゆるやかな離脱（Churn）に対しては、隣人紹介（PEX）とトラッカー（シグナリングサーバー）による補完で、ネットワークはほぼ永久に維持される。
- **一斉離脱への耐性**: 全体の半分が一度に消えても、9割以上のユーザーは大陸（巨大コンポーネント）に留まり、数秒で網を再統合できる。

## 2. ネットワーク分断（孤立・島化）の確率論的分析

### 2.1 個人の孤立確率
ユーザーが繋がっている全隣人 $d$ 人が、再接続（4〜5秒）を試みる前に同時に消滅する確率 $P_{iso}$。
離脱率を $r$、接続数（Degree）を $d$ とすると：
$$P_{iso} = r^d$$

- **日常（離脱率10%）**: $0.1^6 = 1,000,000$ 分の 1。ほぼ無視できる。
- **パニック（離脱率50%）**: $0.5^6 = 0.015$ (1.5%)。数千人規模であれば、数十〜百人程度の短期的な孤立が発生するが、トラッカーへの駆け込み（Re-Signaling）で即時に解消可能。

### 2.2 ネットワークの島化（スプリット・ブレイン）
巨大な網が「日本島」「アメリカ島」のように真っ二つに分断されるリスク。

- **ランダムグラフ理論 (Erdős–Rényi)**: $d \ge 2 \log n$ 程度の接続があれば、ネットワークは統計的に「一つの巨大な大陸（Giant Component）」になる。
- **島化の唯一の原因（クリーク）**: 似たもの同士（地域、興味）で固まってしまい、島同士を繋ぐ「架け橋の数」が極端に少なくなること。
- **対策**: **ランダムウォーク撹拌（Shuffling）**。定期的に既存の接続を切り、全く無関係な「友達の友達」と繋ぎ直すことで、トポロジーの偏りを物理的に破壊し続け、大陸を一つに固定する。

## 3. 「Degree=6」の黄金比

ブラウザのWebRTCという非力な環境における設計指針。

1. **負荷分散**: 全員と繋がらず（フルメッシュ回避）、たった「6人」への負荷に絞る。
2. **高速探索**: 「六次の隔たり（Small World）」により、6人という疎結合でも、1万人の網の中を4〜5ホップでデータが駆け巡る。
3. **偏りの排除**: 上限（Max Degree=12）を設けることで、一部のブラウザが「ハブ」になりすぎてパンクすることを防ぎ、かつ権力を分散（Decentralization）させる。
