### 分析

- **連結性**: 全スケールで完全連結を維持 ✅
- **孤立ノード**: ゼロ ✅
- **⚠️ 問題: 平均経路長の線形増加**
  - 5000ノードで平均463ホップ → **ゴシップ到達遅延が深刻**
  - 原因: 新ノードが「最近追加されたノードの近傍」にしか接続できず、
    グラフが **鎖状（チェーン型）** に成長している
  - **対策**: 撹拌（Shuffle）が不可欠。シナリオ5で改善を確認済み。

## シナリオ2: ランダムノード削除（1000ノードから）

| 削除率 | 削除後孤立 | 修復後連結 | 修復後avg度数 | 修復後avg経路 |
|:------:|:----------:|:----------:|:------------:|:------------:|
| 10% | 0 | ❌(2成分) | 7.22 | 65.82 |
| 30% | 0 | ❌(10成分) | 6.03 | 22.38 |
| 50% | 1 | ❌(8成分) | 6.12 | 6.93 |

### 分析

- **⚠️ 問題: 修復後もネットワークが分断（複数成分）のまま**
- 原因: 現在の`repairAll()`はPEX（隣人の隣人）のみで修復を試みるが、
  チェーン型グラフが分断されると、PEXでは到達できない「島」ができる
- **対策**: repairAll()に「トラッカーフォールバック」をシミュレーションに追加すべき
  - 現実の実装では、PEXで修復できない場合にトラッカーに一時再接続するため、
    分断は解消される
  - 加えて、撹拌によるランダム接続が「小世界性」を付与し、分断耐性が劇的に向上する

## シナリオ5: 撹拌効果の検証（500ノード）

| 状態 | avg度数 | σ | avg経路 | 直径 | CC |
|:----:|:-------:|:---:|:-------:|:----:|:------:|
| 撹拌前 | 7.97 | 0.36 | 45.57 | 138 | 0.7668 |
| **撹拌10ラウンド後** | **7.99** | **0.09** | **6.69** | **13** | **0.1225** |

### 分析

- **撹拌は劇的な効果を発揮** 🎯
  - 平均経路長: 45.57 → **6.69**（85%削減）
  - 直径: 138 → **13**（90%削減）
  - 度数σ: 0.36 → **0.09**（均等性向上）
  - クラスタリング: 0.77 → 0.12（過度なクラスタ解消）
- **撹拌はネットワークの「小世界性」を付与する最重要メカニズム**
  - Watts-Strogatz モデルの「ランダム再配線」と同等の効果

## 最終結論と推奨パラメータ

### 推奨パラメータ（確定値）

```typescript
export const MESH = {
  MIN_DEGREE: 5,
  MAX_DEGREE: 8,
  TARGET_DEGREE: 6,
  SHUFFLE_INTERVAL: 120_000,  // 2分間隔で撹拌
  SHUFFLE_DROP_COUNT: 1,
} as const;
```

### 根拠

1. **min=5, max=8**: 全シナリオで孤立ノード0を達成
2. **target=6**: 接続度の均等性（σ=0.10）が最も優れる
3. **撹拌は必須**: 撹拌なしだとチェーン型成長で経路長が爆発する
4. **Churn耐性**: 5%/ステップの入替でも完全連結を維持
